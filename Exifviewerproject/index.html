Features

Quiet by design. Thorough by default.

ExifViewer doesn't try to be a photo editor. It's a reader. It opens your file, shows you everything it knows, and gets out of the way.

i

Full EXIF read

Camera, lens, settings, GPS, orientation, software, color profile, timestamps — every tag the file carries.

◇

Native SwiftUI

Built for macOS from the ground up. Launches instantly, handles HEIC / RAW / JPEG / PNG without complaint.

⌂

Local-only

Nothing ever leaves your machine. No analytics, no telemetry, no "cloud sync" you didn't ask for.

☼

Drop-in friendly

Drag a photo onto the window, open from Finder, or set it as your default EXIF viewer. It just opens.

⎘

Copy anything

Every field is copy-paste. Grab the GPS coords, paste them into Maps. Grab the timestamp for a caption.

∅

Zero fluff

No accounts. No upsells. No onboarding flow. You open the app, you see the data. That's the whole thing.

Use Cases

When you'd actually reach for it.

Four situations where ExifViewer earns its keep on your dock.

01

Verifying what you shot

Did your camera actually write the right ISO? Was that lens really at 50mm? Check the file, not your memory.

02

Finding where a photo was taken

Photos with GPS tags show exact lat/long. Copy the coords, drop them into Maps, and you're there.

03

Privacy audits before sharing

See exactly what your photo reveals about you — location, device, software — before posting or sending.

04

Checking if a photo was edited

Software tags, modification dates, and color profile often give away whether a file has been through an editor.

Installation

Build it yourself. Four steps.

ExifViewer ships as source. You need a Mac with Xcode. Takes about two minutes end to end.

i.

Install Xcode

Grab Xcode from the Mac App Store (or xcode-select --install if you only need the command line tools and want to use a newer Xcode build later).

ii.

Clone the repository

Pull the source down from GitHub.

$ git clone https://github.com/yourname/ExifViewer.git $ cd ExifViewer/Exifviewerproject
iii.

Open in Xcode

Double-click ExifViewer.xcodeproj, or from the terminal:

$ open ExifViewer.xcodeproj
iv.

Build & run

Pick the ExifViewer scheme, hit ⌘R. First build takes ~30 seconds. After that, drag the app to /Applications if you want it to stick.

# or build from CLI: $ xcodebuild -project ExifViewer.xcodeproj -scheme ExifViewer -configuration Release

FAQ

Small answers to small questions.

What file types does it support?
JPEG, HEIC, PNG, TIFF, and most common RAW formats that macOS can already read. Basically anything Preview opens will also open here, and EXIF from any of them will render.
Do I need a developer account?
No. You build and run locally in Xcode, which signs it with a free personal signing identity. No paid Apple Developer membership required for personal use.
Why isn't there a notarized .app release?
Notarization needs a paid developer account and I'd rather keep it free and open-source. Building from Xcode takes under a minute.
Does it edit or remove metadata?
No — it's read-only by design. If you want to strip EXIF, macOS Preview's Tools → Show Inspector can already do that. ExifViewer is for inspecting, not scrubbing.
Does it send anything over the network?
Not a byte. The app doesn't bind a network socket. Check Console.app or Little Snitch if you want to verify.
Will there be an iOS version?
Maybe. The parsing code is shared SwiftUI/Apple-frameworks, so porting is mostly UI work. If enough people actually use the Mac version, it'll happen.
