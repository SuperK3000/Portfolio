Features

Small script. Big lift.

Sub-100-line Python script with a clear job: watch a folder, and whenever a new folder of photos shows up inside it, bundle them into a slide deck. Zero UI, zero drama.

01 / WATCH

Filesystem watchdog

Built on the excellent watchdog library. Reacts the instant a new subfolder lands, no polling, no cron.

02 / AUTO

Auto-deck generation

Each photo gets its own slide, scaled to fit while preserving aspect. Output lands next to the photos, named after the folder.

03 / SKIP

Idempotent by default

Already has a .pptx? It's skipped. Rerun the watcher on an existing directory tree without fear.

04 / DELAY

Drop-settle window

Waits 10 seconds after a folder appears before bundling, so you have time to finish dragging files in.

05 / LOCAL

Fully local, zero deps

Three Python deps: watchdog, python-pptx, Pillow. Nothing goes off-machine. Nothing calls home.

06 / OPEN

Hackable script

One file, ~90 lines. Change the extension filter, tweak the layout, swap PowerPoint for Keynote — it's all right there.

How it works

Four stages. One pipeline.

Here's what actually happens end to end when a new folder appears.

01 / DETECT
I.

Folder appears

Watchdog fires on_created as soon as a new subfolder lands in the watched directory.

02 / WAIT
II.

Settle window

Script waits 10s so your drag-and-drop finishes. Partial folders don't get half-bundled.

03 / BUILD
III.

Bundle photos

Scans for .jpg / .jpeg / .png, scales each to fit a slide, places centered, saves a deck.

04 / DONE
IV.

Deck appears

A foldername.pptx file drops in next to your photos. Open it. Send it. Done.

Installation

Three commands. You're recording.

You need Python 3.10+. Works on macOS, Linux, and Windows.

01

Grab the script

Download photo_watcher.py and requirements.txt directly, or clone the whole project.

$ git clone https://github.com/yourname/photo-watcher.git $ cd photo-watcher
↓ photo_watcher.py ↓ requirements.txt
02

Install dependencies

Create a venv (recommended) and install the three packages.

$ python3 -m venv .venv $ source .venv/bin/activate # on Windows: .venv\Scripts\activate $ pip install -r requirements.txt
03

Run it

Point it at the folder you want watched. Any new subfolder inside that path will be auto-bundled.

$ python photo_watcher.py ~/Desktop/Photo-Dropoff Scanning existing subfolders in /Users/you/Desktop/Photo-Dropoff... Watching /Users/you/Desktop/Photo-Dropoff for 5s (Ctrl+C to stop early)

Note: by default the watcher runs for 5 seconds then exits — bump RUN_SECONDS at the top of the script to keep it running longer (or set to a large number to leave it on all day).

FAQ

The common questions.

What photo formats are supported?
Out of the box: JPG, JPEG, and PNG. Pillow handles far more than that — HEIC, TIFF, WEBP — if you just add the extension to the PHOTO_EXTS set at the top of the script (and install pillow-heif for HEIC).
What happens if I drop photos into an already-processed folder?
Nothing, by design. The script checks for an existing .pptx with the folder's name and skips. Delete that file to force a rebuild.
Why does it stop after 5 seconds?
The default RUN_SECONDS = 5 was set for quick one-shot runs. Change it to something high (e.g. 86400) to leave it running as a background daemon.
Does it process nested subfolders?
Only direct children of the watched folder. Nested ones are ignored — by design, so you can organize in whatever deeper hierarchy you want without accidentally triggering builds.
Can I change the slide layout?
Yes — it's vanilla python-pptx. The script uses the blank layout and centers each image. Swap in a title layout, add captions with EXIF data, or style per your brand.
Does it work on Windows?
Yes. Same three dependencies, same command. Just remember to use .venv\Scripts\activate instead of source .venv/bin/activate.
